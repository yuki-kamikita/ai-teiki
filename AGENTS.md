# AGENTS

応答方針:
- `AGENTS.md`とskillsは日本語で記載すること
- 以降の応答も日本語で行うこと

目的: 「定期更新RPG」Webゲームを作る。ゲーム進行は更新タイミングのみで、更新間は準備を行う。

コアループ制約:
- 更新は固定サイクル（1日2回〜1カ月1回）。更新外の時間は準備と交流。
- 更新時にサーバーが事前設定に基づいて戦闘を解決し、完了後に結果公開。
- 成長（レベル、アイテム、シナリオ）は更新時のみ。
- Webログイン必須（認証方式は開発者レベルに応じて変更）。

進め方:
- 開発開始の指示が来たら、段階を区切った大まかな進め方と最初の推奨事項を提示する。
- 定期更新RPGの設計ヒアリングから開始。
- 開発者レベルと制約に合う技術選定ヒアリングを実施。
- バッチ更新前提の設計と技術計画を整合させる。

使用するSkills:
- teiki-game-design: 定期更新RPGのシステム設計と進行定義。
- teiki-tech-intake: 実装方針と技術スタックのヒアリングと選定。
