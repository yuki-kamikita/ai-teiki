# 定期更新RPGの設計

定期更新RPGのコア設計を作成・改訂するときに使う。

## クイックスタート

1. 更新サイクル範囲と更新間の想定プレイ時間を確認する。
2. 準備フェーズの行動を定義する（編成、取引、チャット、偵察）。
3. 更新フェーズを定義する（戦闘解決、報酬、シナリオ進行、状態更新）。
4. 成長要素を定義する（ステータス、アイテム、物語の解放条件、長期目標）。
5. ループを簡潔な仕様にまとめる。

## 開発開始時の提案（初回ガイダンス）

開発開始の指示が来たら、まず以下を提示する。

1. 大まかな進め方（段階）
   - 必要機能の列挙と優先順位付け
   - 技術選定（習熟度に合わせた最小構成から開始）
   - 個別機能の実装（更新処理・戦闘・ログイン・交流）
   - ゲームデータ作成（世界観、シナリオ、スキル、敵）
   - 運用準備（更新スケジュール、監視、バックアップ）
2. 最初の推奨事項
   - 「更新バッチの最小実装」を先に決める（更新の入力/出力/ログ）。
   - 「結果表示の最小画面」を先に用意する（更新結果の確認体験）。
   - 早期に小さなプレイサイクルを回し、設計を固める。
   - セキュリティとデータ保全の最低ラインを決める。

## 出力フォーマット

以下の見出しで簡潔にまとめる:
- 更新サイクル
- 更新間の活動
- 更新時の解決内容
- 成長と報酬
- ソーシャル要素
- リスクと未決事項
